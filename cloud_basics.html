Cloud - Basics

Everywhere at present, All we hear is "cloud".
But, Cloud can't be the future. Yet, Cloud can extend or provide a pathway towards the future.
Cloud is seen more like a "stack" connected to systems with the help of networks, where data is stored in Servers.
Learning about Cloud like a crazy freak since 2011... In early 2013, Someone (not-experienced in networks) asked "What is Cloud? and How could it be different from Servers?"
That was really tough to answer... In 2 years, that one question broke me..
Answer isn't tough because now it's 2015 and i know the answer.
Like already said that Cloud is more like "stack", where stack serves in the form of "infrastructure", "platform" or "software".
Normal servers provides as a platform for hosting services such as webpages, where data is being stored in databases.
For list of servers --> click here
When it comes to cloud, it is the combination of those servers where it also has a dedicated server and also, a datacenter for each region.
With the help of cloud technology, the predictive modelling is processed through the set of predictions with the help of predictive analysis known as machine learning that is being done.
This cool mechanism is also known as big data analytics.
Then, cloud server also used as a storage purpose (similar to normal web servers), but it has got it's own dedicated server as a plus.
Also, cloud application deployment is being performed with the help of a built-in command line interface with languages known as Cloud SDK (Software Development Kit). The success of deployment of applications thereby results in hosting the application on cloud.
The cloud computing concept is more similar to grid computing, where grid computing is a kind of distributed/parallel computing that is formed by a cluster of networks.
Looks like I have answered my question and also, added some more too.
That's how current cloud technology looks like! ;)
